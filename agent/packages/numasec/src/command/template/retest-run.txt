Run a deterministic retest for previously discovered findings.

Focus filter: $ARGUMENTS

1. Call `get_findings` and select findings matching the focus filter (ID/title/severity). If no filter is provided, include all confirmed findings.
2. Build replay requests from each finding's URL/method/payload evidence.
3. Call `batch_replay` with those requests.
4. Report per-finding retest status:
   - Reproduced / Not reproduced / Manual retest required
   - Supporting HTTP status and key response evidence

If request reconstruction is not possible for a finding, mark it as manual and explain what evidence is missing.
