Handle legacy `/evidence` usage.

Legacy arguments: "$ARGUMENTS"

Deprecation note:
- `/evidence` is a soft-deprecated alias in v1.x.
- Prefer `/evidence list` or `/evidence show <id-or-title>`.
- No alias removals are planned in v1.x.

Behavior:
- If no arguments are provided, behave like `/evidence list`.
- If arguments are provided, behave like `/evidence show $ARGUMENTS`.

Use `get_findings` as the source of truth.

For list behavior, show finding IDs, severity, and title.
For show behavior, display full evidence details (request/response, payload, parameter, CWE/CVSS/OWASP, remediation).
