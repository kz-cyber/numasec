Create a prioritized remediation plan from current findings.

Call `get_findings` first.

Then produce:
1. Immediate actions (0-7 days) for critical/high findings
2. Short-term hardening (within 30 days)
3. Long-term improvements (architecture/process)

For each action include:
- Vulnerability or root cause addressed
- Concrete remediation steps
- Owner suggestion (app/devops/security)
- Verification criteria for retest

If findings are empty, provide a preventive hardening plan based on OWASP Top 10 controls.
